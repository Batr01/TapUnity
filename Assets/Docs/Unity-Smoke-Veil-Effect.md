# Дым «Smoke Veil» в Unity: префаб и красивое появление / уход

В матче эффект создаётся кодом `MatchController.EnsureSmokeVeilOverlay()`: объект вешается на `PlayArea` (RectTransform), якоря случайные — **корень префаба обязан иметь `RectTransform`**. Если в инспекторе у `MatchController` поле **Smoke Veil Effect Prefab** пустое, остаётся запасной вариант с полупрозрачным `Image` (квадрат). После назначения префаба квадрат заменяется на ваш эффект.

Код для всех дочерних `ParticleSystem` включает **`main.useUnscaledTime = true`**, затем `Clear` + `Play`, чтобы дым не «замирал» при паузе по `timeScale` и совпадал с таймером скилла.

---

## Правильная настройка префаба (порядок действий)

Делайте всё в **Prefab** mode (двойной клик по `SmokeVeilRoot` в Project).

### A. Иерархия и корень

| Что | Как должно быть |
|-----|------------------|
| Структура | **Один** корень `SmokeVeilRoot` с **RectTransform**. Дети — например **`SmokeParticles`**. **Не** вкладывайте второй объект с именем `SmokeVeilRoot` внутрь себя (на скрине у вас как раз так: эффект почти не виден). В рантайме код **сам сплющит** эту ошибку (`SmokeVeilOverlayRuntime.FlattenNestedSmokeVeilRootIfPresent`), но в префабе лучше удалить лишний уровень вручную. |
| Корень | **Layer**: **UI** (как у остального интерфейса). **Anchors** в префабе могут быть stretch на весь родитель — в матче якоря перезапишет код. **Важно:** при stretch-якорях не оставляйте в префабе огромные отрицательные **Width/Height** в **Rect Transform** (например −980 / −1880) — зона дыма сжимается до полоски и эффекта не видно. Для превью держите **0 / 0** или уберите отступы; в рантайме `MatchController` теперь сбрасывает `sizeDelta` при выставлении якорей. |
| Дочерний эмиттер | Объект с **Particle System** лучше тоже поставить на **Layer: UI**. **Position** (0, 0, 0), **Scale** (1, 1, 1). |

### B. Particle System — Main

- **Simulation Space** = **Local** (дым едет вместе с прямоугольником зоны).
- **Scaling Mode** = **Hierarchy** (масштаб от `RectTransform` зоны).
- **Play On Awake** = **выкл** (скилл вызывает `Play()` из кода).
- **Looping** = **вкл** (или длительность системы ≥ времени скилла).
- **Start Lifetime**, **Start Speed**, **Start Size** — подберите так, чтобы в **Scene** при симуляции было видно облако; **Start Speed** держите небольшим (частицы не «улетают» из якорной зоны).

### C. Particle System — Renderer (обязательно проверить)

- **Max Particle Size** = **0** (без лимита на экране). Значение **0.5** при большом **Start Size** делает дым **почти невидимым** — типичная ловушка.
- **Render Mode** = **Billboard** (по умолчанию нормально).
- **Material** — для URP подходит **Particles/Unlit** из пакета URP (или свой материал с альфой и мягкой текстурой дыма).

### D. Внешний вид (по желанию)

- **Color over Lifetime**: альфа **0 → пик → 0** по жизни частицы — мягкое появление и растворение.
- **Size over Lifetime** — мягкое «раздувание».
- **Shape** (Cone / Box и т.д.) и **Rate over Time** — чтобы заполнить прямоугольник зоны без дыр.

### E. Сцена и матч

- **Онлайн vs тренировка:** скиллы 2–3 в **онлайне** по задумке приходят **с сервера** на ваш клиент только когда срабатывает событие (другой игрок вас «задебаффил»). Своей кнопкой «Дымовая завеса» вы шлёте эффект **сопернику** — **на своём экране дым не появится**, пока нет второго клиента / ответа сервера. В **тренировке** эффект применяется **сразу у вас** (локально). Для проверки префаба в онлайне без пары включите у `MatchController` опцию **Debug Apply Opponent Visual Skills Locally After Online Send** (в сцене **Match** она уже включена по умолчанию; если сервер дублирует событие — выключите, чтобы эффект не накладывался дважды).
- **Окно Game в Prefab Mode** часто **не показывает** обычный `Particle System` так же, как полноценная сцена: для проверки откройте вкладку **Scene** (Shaded) или запустите сцену **Match** в Play Mode / используйте **Smoke Veil Local Preview** на `PlayArea`.
- **Screen Space — Camera + Hierarchy scaling:** у зоны дыма в мире часто крошечный `lossyScale`, частицы почти не видны. В `MatchController` при спавне вызываются `CompensateSmokeVeilScaleIfCanvasMakesParticlesTiny` и подложка `SmokeVeilBackdrop` (полупрозрачный `Image`). Если видна **только подложка** — настраивайте частицы/материал; если **ничего** — смотрите якоря/`PlayArea` и что скилл реально вызывается.
- В **Match** / **Training** корневой **Canvas** должен быть **Screen Space — Camera** с **Main Camera** (в вашем проекте так настроено) — иначе частицы под чистым Overlay часто не видны.
- На объекте с **`MatchController`** в поле **Smoke Veil Effect Prefab** перетащите **`SmokeVeilRoot`**.
- Проверка без скилла: компонент **`Smoke Veil Local Preview`** на `PlayArea`, клавиша **P** (Input System).

Ниже те же темы разобраны подробнее.

## 1. Подготовка

1. Откройте проект в Unity (у вас Unity 6).
2. В **Project** создайте папку, например `Assets/Prefabs/UI/`.
3. На сцене с матчем найдите объект с **`MatchController`** — после создания префаба вы перетащите его в поле **Smoke Veil Effect Prefab**.

---

## 2. Корень префаба (обязательно UI)

1. **Hierarchy** → ПКМ → **UI** → **Empty** (или создайте пустой объект под существующим Canvas и перенесите под `PlayArea` для визуальной проверки).
2. Переименуйте, например, `SmokeVeilRoot`.
3. Иерархия префаба: **один корень** с `RectTransform` и **прямые дети** вроде `SmokeParticles` — не вкладывайте второй объект с тем же именем `SmokeVeilRoot` внутрь себя (в инспекторе это выглядит как дубль и путает отладку).
4. Убедитесь, что на корне есть **`RectTransform`** (у объектов UI он есть по умолчанию).
5. **Anchors**: временно **Stretch** на весь родитель — для отладки; в бою якоря выставит код, префаб всё равно должен нормально тянуться от якорей.
6. При желании добавьте на корень **`CanvasGroup`** (альфа всего слоя при снятии скилла код пока не анимирует — только для ручных тестов или будущих доработок).

Перетащите объект из Hierarchy в `Assets/Prefabs/UI/` — получится **префаб**. Дальше правьте префаб в **Prefab** mode.

---

## 3. Частицы: базовый дым

1. Под корнем создайте **ПКМ** → **Effects** → **Particle System** (дочерний объект), имя например `SmokeParticles`.
2. **Transform**: `localPosition` (0, 0, 0), масштаб при необходимости подгоните под зону (часто 1,1,1 достаточно, размер задаётся модулями).

### Main module

- **Duration**: чуть больше длительности скилла на клиенте (`opponentSmokeVeilDurationSec`, по умолчанию 5 с), либо **Looping** включён — тогда дым идёт всё время, пока объект не уничтожат.
- **Start Lifetime**: 1.5–3.5 — дольше = более «мягкий» хвост.
- **Start Speed**: небольшой (0.1–0.8), иначе дым улетит из зоны.
- **Start Size**: с **Random Between Two Constants** или кривая — лёгкая вариация.
- **Start Color**: серый с невысокой альфой; детальную альфу лучше вести в **Color over Lifetime**.
- **Simulation Space**: **Local** — дым двигается вместе с прямоугольником зоны.
- **Scaling Mode**: **Hierarchy** — масштаб родителя влияет на частицы (удобно под UI).
- **Play On Awake**: можно выключить — скилл всё равно вызывает `Play()` из кода.

### Emission

- **Rate over Time**: 15–50 (подберите по FPS и вкусу).
- Опционально **Bursts** в начале для «вспышки» появления.

### Shape

- **Box** или **Rectangle** (в зависимости от версии), размер по X/Y сопоставьте с типичным якорным прямоугольником (код задаёт ширину ~0.32–0.58 и высоту ~0.36–0.62 от родителя — это доли `PlayArea`). Можно оставить широкий box и ослабить **Rate**.

### Renderer

- **Max Particle Size** (в `Particle System Renderer`): если стоит **0.5** и при этом большой **Start Size**, Unity **ужимает** частицы на экране до долей пикселя — дым почти не виден. Поставьте **0** (= без ограничения по экрану) или большое значение (например **200**).
- **Render Mode**: **Billboard** (классика для дыма).
- Материал: под ваш пайплайн (Built-in / URP) используйте шейдер частиц с альфой, например вариант **Particles/Standard Unlit** или URP-аналог с прозрачностью.
- **Sorting**: если дым рисуется **под** кругами, увеличьте **Order in Layer** у `ParticleSystem Renderer` или оберните частицы в дочерний **Canvas** с **Override Sorting** и высоким **Sort Order** (см. раздел 5).

### Красивое появление и уход

1. Включите **Color over Lifetime**: градиент по альфе **0 → пик (например 0.35–0.6) → 0** по времени жизни частицы — мягкое появление и растворение.
2. Включите **Size over Lifetime**: кривая **мало → больше → мало** или плавный рост — дым «раздувается» и растворяется.
3. Опционально **Velocity over Lifetime** / **Noise** — лёгкое завихрение (сила маленькая, иначе хаос).
4. **Rotation over Lifetime** — очень слабая случайная вращательная скорость для живости.

Согласуйте **Start Lifetime** и альфу в градиенте так, чтобы к моменту снятия скилла картинка не обрывалась резко (или оставьте **Looping** + короткую жизнь частицы — тогда при `Destroy` эффект просто исчезнет; код при уничтожении вызывает `Stop` с очисткой).

---

## 4. Текстура дыма

1. Импортируйте мягкое облако / soft smoke (PNG с плавным краем).
2. **Texture Type**: при необходимости **Sprite (2D and UI)** или настройки для частиц по гайду вашего шейдера.
3. В модуле **Renderer** назначьте материал с этой текстурой; включите **Soft Particles** при рендере в глубину, если используете перспективную камеру (для чистого UI Overlay часто не критично).

---

## 5. Если дым не виден или под кругами

- **Screen Space — Overlay** и частицы: обычный `ParticleSystem` под **только Overlay** часто **не виден** (UI и мир рисуются по-разному). В сценах **Match** и **Training** корневой игровой **Canvas** переведён в **Screen Space — Camera** с **Main Camera** и **Plane Distance 100** — тогда дым и круги проходят одной цепочкой с камерой, эффект должен быть виден без костылей.
- **Запасной путь** (другая сцена с чистым Overlay): при спавне `MatchController` может создать дочерний **`SmokeParticleCanvas`** (`SmokeVeilOverlayRuntime`) с **Screen Space — Camera** и **`Camera.main`**. Нужен тег **MainCamera** у камеры.
- **Overlay UI + мир** (сортировка): частицы по умолчанию могут сортироваться не так, как `Image` на Canvas. Варианты:
  - Поднять **Order in Layer** у рендерера частиц выше, чем у элементов игрового поля.
  - Либо под корнем префаба: дочерний объект с **Canvas** (Render Mode как у основного игрового Canvas), **Override Sorting** = On, **Sort Order** выше, чем у слоя с кругами; частицы — дочерние к этому Canvas.
- Проверьте **Mask**: если `PlayArea` обрезает дочерние элементы маской, частицы должны быть в той же иерархии, что и остальной контент, который маска затрагивает.

---

## 6. Проверить эффект отдельно (без скилла)

1. Откройте сцену **Match** (нужны **Main Camera** и Canvas в режиме **Screen Space — Camera**, как в проекте).
2. На объекте **`PlayArea`** (или другом с `RectTransform` под Canvas) нажмите **Add Component** → найдите **`Smoke Veil Local Preview`** (`TapBrawl.DebugTools`).
3. В поле **Smoke Veil Prefab** перетащите **`SmokeVeilRoot`**.
4. Запустите **Play** и нажмите клавишу **`P`**: в инспекторе это поле **Preview Key** (`UnityEngine.InputSystem.Key`, по умолчанию `P`). В центре зоны появится превью на несколько секунд; в консоли будет строка `[SmokePreview]`.
5. Клавишу и время жизни можно поменять в инспекторе. После проверки компонент можно удалить.

Превью использует **новый Input System** (`Keyboard.current`), без класса `UnityEngine.Input` — при активном режиме **Input System Package** в Player Settings исключение не возникает.

Логика Canvas такая же, как у `MatchController` (через `SmokeVeilOverlayRuntime`).

---

## 7. Подключение к игре

1. Сохраните префаб.
2. Откройте сцену **Match** / **Training**.
3. Выберите объект с **`MatchController`**.
4. В поле **Smoke Veil Effect Prefab** перетащите префаб.
5. Запустите сцену, активируйте скилл дыма — должны играть частицы в случайной области `PlayArea`.

Если в консоли ошибка про **RectTransform**, добавьте его на корень префаба (объект должен быть UI или иметь компонент, дающий `RectTransform`).

---

## 8. Краткий чеклист

| Шаг | Действие |
|-----|----------|
| 1 | Корень = один `RectTransform`, без дубля `SmokeVeilRoot` внутри |
| 2 | Layer **UI** на корне и на объекте с частицами |
| 3 | Main: **Local**, **Hierarchy**, **Play On Awake** off, **Looping** или длительность |
| 4 | Renderer: **Max Particle Size = 0**, материал с альфой (URP Particles Unlit) |
| 5 | Color / Size over Lifetime — мягкая альфа и размер |
| 6 | При необходимости **Order in Layer** у рендерера, если дым под кругами |
| 7 | Префаб в **MatchController** → **Smoke Veil Effect Prefab** |

После этого квадрат из `Image` больше не используется, пока поле префаба заполнено.
